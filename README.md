# Klaus Korrekt — Telegram Language Practice Bot

Небольшой Telegram-бот для разговорной практики с голосовым ИИ-собеседником.
Пользователь отправляет voice-сообщение, бот исправляет ошибки одной компактной
строкой и продолжает разговор текстом и/или голосом.

## Возможности

- языки: Deutsch, English, Norsk (bokmål) и Español;
- уровни CEFR от A1 до C2;
- исправления в формате `ошибка → исправление` внутри одной фразы;
- голосовые ответы модели `openai/gpt-audio-mini` через OpenRouter;
- опциональная текстовая копия фактически произнесённого ответа;
- отдельные настройки и короткая история для каждого пользователя;
- режим Topic с 28 заранее подготовленными ролевыми ситуациями;
- простое журналирование входных сообщений, исправлений и ответов;
- одновременная работа нескольких пользователей без общей истории.

История содержит не более четырёх последних пар реплик. Текстовая копия и аудио
не дублируются в контексте: фактически произнесённый ответ записывается один раз.

## Команды

- `/start` — приветствие и быстрые настройки;
- `/settings` — меню всех настроек с inline-кнопками;
- `/language` — выбрать язык;
- `/level` — выбрать уровень CEFR;
- `/text` — включить или выключить текстовую копию ответа;
- `/topic` — начать случайную ролевую ситуацию;
- `/reset` — очистить историю и завершить активный Topic.

Некоторые команды принимают аргументы напрямую:

```text
/language de
/language en
/language no
/language es
/level B1
/text on
/text off
```

При смене языка, уровня или Topic предыдущая история очищается. `/reset` сохраняет
выбранные язык, уровень и настройку Text. Все данные хранятся только в памяти и
сбрасываются после перезапуска процесса.

## Как обрабатывается сообщение

1. Telegram voice в OGG/Opus скачивается в память и преобразуется в MP3.
2. OpenRouter распознаёт речь, исправляет её и планирует короткий ответ.
3. Та же модель генерирует PCM16-аудио; конечная тишина обрезается.
4. PCM16 преобразуется в OGG/Opus и отправляется как Telegram voice.
5. Если включён Text, показывается транскрипция реально отправленного аудио.

## Требования

- `ffmpeg` и `ffprobe` в `PATH` если Windows и установленный `sudo apt install -y ffmpeg` если Lunux;
- токен Telegram-бота;
- ключ OpenRouter с доступом к `openai/gpt-audio-mini`.

## Настройка

Создайте `.key` по образцу [.key_example](.key_example):

```text
TG=your_telegram_bot_token
OpenRouter=your_openrouter_api_key
```

Файлы `.key` и `.gitsecret` исключены из Git.

Создайте локальное conda-окружение и установите зависимости:

```powershell
& "$env:USERPROFILE\miniconda3\Scripts\conda.exe" create -p .\.conda python=3.12 -y
& .\.conda\python.exe -m pip install -r requirements.txt
```

## Запуск

```powershell
& .\.conda\python.exe .\bot.py
```

Остановка интерактивного процесса: `Ctrl+C`.

## Проверка

```powershell
& .\.conda\python.exe -m pytest -q
& .\.conda\python.exe -m compileall -q bot.py topics.py
```

## Файлы проекта

- `bot.py` — Telegram handlers, OpenRouter-клиент, настройки и аудиопайплайн;
- `topics.py` — каталог ролевых ситуаций;
- `test_bot.py` — локальные тесты;
- `requirements.txt` — Python-зависимости;
- `logs/bot.log` — журнал запущенного бота, создаётся автоматически.

## Ограничения текущей версии

- настройки и история не сохраняются после перезапуска;
- поддерживаются только Telegram voice-сообщения;
- рассчитано на небольшое число одновременных пользователей;
- для входного и выходного аудио требуется локальный `ffmpeg`.
